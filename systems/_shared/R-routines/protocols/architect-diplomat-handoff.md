# Architect-Diplomat Handoff Protocol

## Purpose

Define the clean handoff between the Architect (Asaf, 0→1) and the Diplomat (MG, 1→100). When the Architect's system/model/prototype is locked, the project moves to field execution. The Architect is intellectually freed to return to the lab — zero guilt.

> From the Realization Manifesto, Rule 4: The Ultimate Handoff.

## The Two Roles

### The Architect (Asaf)
- Vision, strategy, R&D
- Financial modeling and underwriting
- AI system development
- Brand building and narrative crafting
- 0 → 1: invents the machine

### The Diplomat (MG)
- Emotional intelligence and trust-building
- Face-to-face negotiation
- Crisis resolution
- "Holding the room" and closing deals
- 1 → 100: runs the machine at scale

## Handoff Criteria

A project is ready for handoff when ALL of these are true:

### For a Deal / Investment Project
- [ ] Financial model complete with worst-case scenario validated (Truth Engine passed)
- [ ] Pitch deck / investor narrative locked (Trust Engine complete)
- [ ] Legal structure defined
- [ ] Key assumptions documented
- [ ] Risk factors identified and quantified

### For an AI System / Tool
- [ ] Prototype working and tested
- [ ] Documentation complete (what it does, how to use it)
- [ ] Known limitations documented
- [ ] Integration points clear

### For a Venture Architecture
- [ ] Venture architecture document complete
- [ ] Sprint 0 diagnostic passed (Go recommendation)
- [ ] Budget model validated
- [ ] Licensing pathway mapped
- [ ] Partner requirements identified

## Handoff Ceremony

1. **Architect creates handoff brief** — one-page summary: what was built, key decisions made, what the Diplomat needs to know, expected next steps
2. **Architect flags handoff ready** — updates state-map.md with handoff status
3. **Diplomat reviews** — asks any clarifying questions
4. **Handoff confirmed** — Architect is freed from the project
5. **Architect moves to next sprint** — no guilt, no looking back

## After Handoff

- The Architect does NOT get pulled back into day-to-day operations
- If the Diplomat encounters a strategic question that requires the Architect's input, it should be a bounded consultation (30 minutes max), not a re-engagement
- The Architect's state-map should show "handed off" status for completed projects

## Key Principle

The Architect's value is in building new pipes, not maintaining old ones. Every hour spent in operational treadmill work is an hour not spent on the next breakthrough. Protect the Architect's creative and strategic bandwidth.
